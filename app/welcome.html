Free for All

This is an open and free repository for all sorts of digital things. Digital anchors for real world objects and encoded assertions from real people are stored here without prejudice. Our mission is to eliminate the obstacles preventing technological solutions to scholarly questions allowing small institutions or independent researchers to start work with minimal resources. The natural state of digitized knowledge should be interoperable, accessible, encoded, and open.

The Roots of Rerum

Rerum (Latin: "of things") is a conveniently ambiguous umbrella term we use to identify the things (tools, schemata, databases) that are functional or philosophical kin. The LUCA concept for them all is a set of principles that guides the development of this ecosystem:

Annotation is everywhere
Scholarly assertions, descriptive annotations, measurements, and notes often mistaken for metadata are vertices for standards. These declarations should be stored as atoms, especially when they are simple to constellate for user applications.
Attribution is essential
Productive communication between researchers requires reliable attribution. With attributed annotations, collaboration becomes natural, and even complex concerns such as authorization are just acts of filtering.
Format is ephemeral
No one can predict the Next Great Format, but know some obstacles to migration can be avoided. The simpler it is to abandon, the more likely a format will remain accessible.
Knowledge is controversy
Statements of fact are reference material until challenged. The collisions of theories, evidence, and educated conjecture or theses enlivens research. Recording both the state and the history of the conversation serves knowledge.
Research is decentralized
Experts are distributed across institutions. The most influential investigators move around, as people do. The complex conventions around citation and rights make clear the challenge of maintaining attribution and encouraging distribution.

The Leaves of Rerum

The Rerum ecosystem includes several distinct categories of tools and services:

RERUM
is any application instance that runs a repository with the Rerum API.
Rerum API
collects the set of HTTP(S) endpoints used by client servers to run operations within a RERUM service.
RERUM Server
is a complete web service. Our existing stack is RHEL-MongoDB-NodeJS.
Rerum Components
are the applications, tools, and Web Components that are designed for the Rerum API.
Rerum Inbox
is the Rerum LDN Inbox announcements reference implementation. It does not utilize the same datastore or API as RERUM, but is a functional and philosophical sibling.
